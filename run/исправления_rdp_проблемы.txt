# ИСПРАВЛЕНИЯ ПРОБЛЕМ С RDP И INPUT CAPTURE WINDOW

## ПРОБЛЕМА
Программа иногда не может скопировать текст в RDP-сессии из-за появления "Input Capture Window" overlay.
Это происходит когда Windows создает оверлей перехвата ввода, который мешает UIA видеть реальные элементы управления.

## КРИТИЧЕСКИЕ ПРИЧИНЫ ПРОБЛЕМЫ

1. **Зажатые клавиши-модификаторы** (Ctrl, Shift, Alt) - keyup события теряются в RDP
2. **Right click + Esc метод** - создает Input Capture Window overlay
3. **Смешивание SendKeys и pyautogui** - разные уровни input-стека создают конфликты
4. **Слишком быстрые клики после фокуса** - RDP требует больше времени для передачи фокуса
5. **UIA ControlFromPoint в RDP** - часто возвращает overlay pane вместо реального элемента

## ВНЕСЕННЫЕ ИСПРАВЛЕНИЯ

### 1. СОЗДАН ФАЙЛ key_utils.py
Содержит критические функции для предотвращения проблем:

- `release_all_keys()` - освобождает все модификаторы (Ctrl, Shift, Alt, Win)

- `stable_copy_text()` - самый стабильный метод копирования для RDP
- `wait_for_overlay_disappearance()` - ожидает исчезновения Input Capture Window
- `disable_problematic_methods()` - отключает методы, создающие overlay

## АЛГОРИТМ СТАБИЛЬНОГО КОПИРОВАНИЯ (RDP)

1. `release_all_keys()` - освободить все модификаторы
2. `wait_for_overlay_disappearance()` - подождать исчезновения overlay
3. `ensure_focus_and_wait(x, y, 0.6)` - установить фокус с ожиданием
4. `safe_hotkey('ctrl', 'a')` - выделить текст с освобождением клавиш
5. `safe_hotkey('ctrl', 'c')` - скопировать с освобождением клавиш
6. `wait_for_clipboard_change()` - дождаться реального изменения буфера

## РЕКОМЕНДАЦИИ ДЛЯ ПОЛЬЗОВАТЕЛЯ

1. **В RDP настройках Windows отключить:**
   - Sticky Keys (Залипание клавиш)
   - Filter Keys (Фильтрация ввода)
   - Toggle Keys (Звуковые сигналы)

2. **Использовать стабильный метод копирования:**
   ```python
   from key_utils import stable_copy_text
   text = stable_copy_text(x, y)
   ```

3. **Избегать:**
   - Right click + Esc комбинаций
   - Смешивания SendKeys и pyautogui
   - Слишком быстрых действий после фокуса
